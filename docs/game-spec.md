# BOKUZUSHI 仕様メモ

最終確認日: 2026-03-07

## 目的

- 派手で気持ちいいが、盤面の可読性と再挑戦のテンポを壊さない。
- モバイルでも視認性と操作性を維持する。
- README と実装のズレを減らすため、数式と状態遷移をここに集約する。

## 変更時の優先順位

1. 数式・状態遷移・しきい値をこの文書と `src/gameRules.ts` で更新する。
2. README の生成元である `src/gameContent.ts` と `scripts/generate-readme.ts` を必要に応じて更新する。
3. `npm run test:run` と `npm run docs:check` を通してから README を確認する。

`docs:generate` / `docs:check` は `node --import tsx` 経由で実行し、制限付き環境でも `tsx` CLI の IPC ソケットに依存しないようにする。

## コアループ

1. パドルでボールを返す。
2. ブロックを削るか、色優位なら貫通破壊する。
3. ヒット数でレベルを上げ、攻撃力と色ティアを伸ばす。
4. 連続破壊で連ツキと沸点ゲージを溜める。
5. 星を破壊してウェーブ突破、全ウェーブ突破で次ワールドを解放する。

## 色と攻撃力

- 色の強さ: `緑 < 青 < 黄 < 橙 < 赤 < 藍 < 紫 < 黒`
- 判定:
  - `ball tier > block tier`: 貫通破壊、反射しない
  - `ball tier <= block tier`: 反射しながらダメージ
- 通常ダメージ:
  - `attackPower = 1 + ballColorTier + floor((level - 1) / 2)`
- 目的:
  - レベルアップしても「色が上がらない区間」で攻撃力が死なないようにする。

## 進行

- 初期レベル: `1`
- 次レベル必要ヒット:
  - `35 + (level - 1) * 20`
- レベルアップ時:
  - ボール色ティアを 1 上げる
  - 最大ティア到達時は黒玉になる

## 沸点 / 連ツキ

- 連ツキ継続時間: `2.0s`
- Gauge は主に連続破壊と貫通破壊で増える
- 沸点発動条件:
  - Gauge が `100` 到達
- 沸騰持続:
  - `8.0s`
- 沸騰効果:
  - 得点倍率 `x2`
  - 演出強化
  - モバイル対応端末では短い振動を返す

## スター配置

- 各ワールドの 3 ステージは、下からおおむね `50% / 70% / 90%` の位置に置く。
- 同じステージ番号でも、後半ワールドほど少しだけ奥へ寄せる。
- 初期面では近めかつ中央寄りに置き、レベル未成長でも到達しやすくする。
- 後半面だけ深い位置と横ずれを強める。
- 序盤は星の下に広めの通路を確保し、終盤ほど通路を細くする。
- 星の近傍ブロックは後半ほど色ティアと HP を上乗せし、突破に段位上げを要求する。

## スコア

- ブロック破壊: `block.maxHp * 10`
- 星破壊:
  - `max(100, 2000 - paddleBounces * 30)`
- 沸騰中:
  - 上記に `x2`

## モバイル表示

- 画面幅が狭い場合は、縦横比よりもゲーム横幅を優先してカメラを広げる。
- coarse pointer ではパドルを上に持ち上げ、指の下に沈みにくくする。
- HUD は薄い 1 行チップ化し、盤面上部の占有を減らす。
- `navigator.vibrate()` が使える端末では、強い報酬イベントだけ短く振動させる。

## 状態遷移

- `start -> playing`
- `playing -> waveclear | worldclear | gameover`
- `waveclear -> playing`
- `worldclear -> playing | start`
- `gameover -> start | playing`
