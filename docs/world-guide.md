# BOKUZUSHI 世界観ガイド

最終確認日: 2026-03-08

## 目的

- ゲーム内文言の設定、語彙、トーンを一つの方向へ揃える。
- 直接的なネット用語やパチンコ実務語を避けつつ、過剰で偏った熱量を持つ独自世界を定義する。
- 今後の UI 文言差し替えやタイトル再検討の基準を先に固定する。

## 作品の芯

この作品は、**刺激が飽和した盤域の深部へ潜る遊戯** である。

プレイヤーは勝者でも勇者でもなく、飽和した気配に引かれて盤域へ沈んでいく **潜行者** である。  
盤面に積み重なる障壁を崩し、奥に沈んだ **奥核** を暴くことだけが、いま目の前にある目的になる。

重要なのは「楽しい冒険」ではなく、**熱、濁り、ざわつき、過剰な気配** である。  
この世界は気持ちよさを説明しない。すでに沸き切った盤の残響だけを見せる。

## トーンの基準

この世界で優先する感覚は次の通り。

- 飽和している
- 熱がこもっている
- 不穏だが美しい
- 何かが近い
- もう少しで奥に届きそう
- 正気より勢いが勝っている

逆に、次の方向には寄せない。

- 現実の SNS やインターネット文化を直接連想させる語
- 実在の遊技機やホール文化を直接参照する語
- SF メカや軍事物のような乾いた硬さ
- 露悪一点張りで読み味が悪くなる造語

## 世界の前提

- 世界は **盤域** と呼ばれる層状の場でできている。
- 各盤域は桜花・炎祭・紅葉・氷雪・黄金のような気配名を持つ。
- 盤域の内部には複数の **層** があり、潜るほど熱量が濃くなる。
- プレイヤーが壊しているのは単なるブロックではなく、流れを濁らせる **障壁** である。
- 各層の奥には封じられた **奥核** がある。潜行者はそれを暴くために盤を穿つ。
- 盤の流れが噛み合うと、一時的に **沸界** に入る。これは祝福というより、盤そのものが過熱している状態である。

## 用語の正規形

| 概念 | 正規語 | 補足 |
| --- | --- | --- |
| プレイヤー | 潜行者 | 盤域の深部へ潜る者 |
| ワールド | 章 | プレイヤー向け表示では「章」を使う |
| ウェーブ | 層 | 章の中の段階 |
| ブロック | 障壁 | 必要に応じて「障壁層」も可 |
| 目標物 | 奥核 | 実装上の `Star` は当面そのままでよい |
| ボール | 燼珠 | 熱を帯びた破砕の珠 |
| パドル | 受け盤 | 珠の流れを返す盤 |
| レベル | 段位 | 潜行者としての到達度 |
| 色ティア | 砕色 | UI では「玉色」よりこちらを優先候補とする |
| Fever | 沸界 | 盤の熱が臨界を越えた状態 |
| コンボ | 連環 | 連続破壊の勢い |
| Game Over | 飲まれた / 沈んだ | 軽い言い回しより、盤に呑まれる表現を優先 |
| Clear | 到達 / 開いた | 奥核や次層に届いた感覚を優先 |

## 文体ルール

- 日本語を主とし、英語は飾りとして短く添える。
- 文章は短く、熱を持たせる。
- 何をすべきかを先に書き、意味づけは後に置く。
- 名詞はできるだけ独自語へ寄せるが、初見で読めない当て字は避ける。
- 語感は「湿っている」「熱い」「濁っている」「近い」を優先する。
- 言い切りは強く、説明は削る。

## 章の扱い

現行の章名は維持する。

- 桜花
- 炎祭
- 紅葉
- 氷雪
- 黄金

これらは季節ステージではなく、盤域ごとの **気配名** として扱う。  
プレイヤーに見せる時は「章」と組み合わせる。

例:

- 桜花 第一層
- 氷雪 第二層
- 黄金 最深層

## UI 文言の置換基準

今後の差し替えでは次を優先する。

| 現行の方向 | 今後の優先語 |
| --- | --- |
| ワールド | 章 |
| ステージ / ウェーブ | 層 |
| 星 | 奥核 |
| 玉色 | 砕色 |
| 沸騰 | 沸界 |
| コンボ / 連ツキ | 連環 |

現時点での推奨例:

- `章を選べ`
- `奥核へ至れ`
- `障壁を穿て`
- `沸界が近い`
- `連環は続いている`
- `次の層へ`
- `飲まれた`

避けたい例:

- `右打ちの顔`
- `当たりの川`
- `保留育ち`
- `先バレ本番`
- `TL の最深部`

## タイトル方針

`BOKUZUSHI` は現在の repo 名・運用名として残してよい。  
ただし、世界観に合わせた表示名へ将来的に改題する余地はある。

現時点の候補:

- **沸界深度**: 最も世界観の芯に近い
- **残響盤**: 語感が強く、短い
- **奥核到達**: 目的が明確

このガイド上の推奨第一候補は **沸界深度** とする。  
実装上のタイトル差し替えは、UI 文言の統一と同じタイミングで行う。

## 実装との折り合い

- `Star` などの既存実装名は、コード互換性を優先して当面そのままでよい。
- ただし、ユーザーに見える文言では「星」ではなく「奥核」を優先候補にする。
- README や仕様書で実装用語と世界観用語が衝突する場合は、世界観ガイドを基準に表示文言を決める。
